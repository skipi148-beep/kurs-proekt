# Отчёт о результатах проведения тестирования

## Краткое описание
В ходе выполнения проекта было проведено автоматизированное тестирование веб-сервиса покупки туров. В качестве СУБД использовался контейнер PostgreSQL 15. Тестирование включало проверку сквозного сценария: UI-взаимодействие -> отправка данных в симулятор банковских шлюзов -> фиксация финального статуса в СУБД.

## Результаты прогона
* **Общее количество тест-кейсов:** 1 (Позитивный сценарий покупки по дебетовой карте APPROVED).
* **Успешные (Passed):** 0
* **Упавшие (Failed):** 1 (100% падений)

## Выявленные дефекты
* **Критический дефект (Bug #1):** При успешной отправке формы дебетовой карты на UI отображается всплывающее окно об успешной оплате (`notification_status_ok`), однако запись в таблице `payment_entity` полностью отсутствует. Данные не сохраняются в СУБД.

## Общие рекомендации
Приложение категорически не рекомендуется к релизу. Требуется исправить интеграцию бэкенда с базой данных PostgreSQL, так как текущая реализация приводит к потере платежных данных клиентов.
