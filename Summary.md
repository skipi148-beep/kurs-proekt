# Отчёт по итогам автоматизации

## 1. Что было запланировано и что реализовано
* **Запланировано:** Развернуть инфраструктуру в Docker Compose, настроить виртуальное окружение Python 3.14, написать базовый UI-тест дебетовой карты Selenium с валидацией записей в PostgreSQL через SQLAlchemy.
* **Реализовано:** Запланированный стек полностью развернут и настроен. Написан и успешно запущен тест с умным ожиданием (polling) базы данных.

## 2. Сработавшие риски и трудности
* **Совместимость с Python 3.14:** Новейшая версия Python потребовала принудительного обновления SQLAlchemy до актуальной версии для устранения ошибок типизации.
* **Драйверы Windows:** Возникла проблема с `webdriver-manager`, которая была успешно решена переходом на встроенный современный механизм Selenium Manager.
* **Асинхронность бэкенда:** Риск падения тестов из-за задержки СУБД был нивелирован созданием цикла опроса базы данных (polling loop).

## 3. Итог по времени
* **План:** 4 часа на настройку окружения и первый тест.
* **Факт:** 5 часов. Небольшое расхождение связано с адаптацией библиотек под Python 3.14 и отладкой сетевых зеркал Docker.
